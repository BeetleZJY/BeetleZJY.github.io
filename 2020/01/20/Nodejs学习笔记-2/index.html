console模块

日志都是默认打印到控制台。最常用的是console.log()、console.error()两个方法。

  • console.log(msg)：普通日志打印。
  • console.error(msg)：错误日志打印。
  • console.info(msg)：等同于console.log(msg)
  • console.warn(msg)：等同于console.error(msg)

console打印一些小技巧输出变量 定义name = ‘jack’。

  • console.log(你好${name} // 输出你好jack
  • console.log(‘你好%s’，name) // 输出你好jack

console.table 打印对象为表格数据，视觉上查看数据比较明了。可输入数组或者对象。

  • console.table({
    id: 0100,
    name: ‘jack’,
    age: 15,
    hobby: ‘movie’
    })
  • console.table([‘apple’，’pear’, ‘banana’])

console.time计时，一般用来测试性能或者请求时间，必须定义同一个tag，单位为毫秒。

console.assert(value, message) 进行断言。

node 10.0版本以上 使用非真的断言调用 console.assert() 只会导致打印 message 到控制台而不会中断后续代码的执行。
node 10.0版本以下 如果value不为true，那么抛出AssertionError异常，并中断程序执行。

console.dir(obj[, options]), 打印大型复杂对象

options 选项， 一般用来打印对象

  • showHidden 如果为 true，则也会显示对象的不可枚举属性和符号属性。默认值: false。
  • depth 告诉 util.inspect() 格式化对象时要递归多少次。这对于检查大型复杂对象很有用。要使其无限递归，可传入 null。默认值: 2。
  • colors 如果为 true，则输出将使用 ANSI 颜色代码进行样式设置。颜色可定制，请参阅自定义 util.inspect() 颜色。默认值: false。

new Console(stdout[, stderr][, ignoreErrors])

一般用来在文本输出打印信息。第一个输出信息，第二个错误打印信息， 如果只有第一个输出选项， 所有调试信息都在第一个文本信息。 如果有第二个选项， 则error和warn信息都会进入定义第二个选项文件。

console.group用于打印一组信息，后续行会往后推后两个

console.trace追踪调用轨迹

此方法可以跟踪函数的调用轨迹。如果需要知道一个函数的调用轨迹，那么可以将此方法写在函数内部即可。
