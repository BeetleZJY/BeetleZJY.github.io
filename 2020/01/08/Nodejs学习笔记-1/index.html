聊一聊javaScript 的前世今生

前言

我们如今看到前端技术蓬勃发展， 前端技术由单单浏览器的交互开发，开始引申到服务器，移动端，桌面端， 大前端的概念的愈发流行。 前端开发天花板不再存在， 能限制你的只是你能不能跟的上前端快速发展的技术。而前端开发也从过去的页面仔，切图仔慢慢也有了自己专属的名称：前端开发工程师，要知道在过去可是根本没有这个职位! 那是什么原因令JavaScript这门语言如今如火如荼， 那么我斗胆按自己个人理解聊聊JavaScript的前世今生。

JavaScript的出生

聊到js这门语言，我们不得不聊下JavaScript之父-Brendan Eich。1995年这位大神还在网景公司(做浏览器)上班，当时网景公司与sun公司(java)合作密切，因而网景公司高层想要有一门类似java脚本和浏览器做一些动态交互。当时领导高层把任务交派给Brendan Eich开发。但是Brendan对java没有兴趣， 为了应付公司任务， 仅仅用了10天就仓促把这门语言给创造出来，当时叫liveScript。毫无疑问10天就出来的js就像个早产儿，先天不足。网景公司推出了这门语言，后面为了蹭一波java的热潮，更名叫JavaScript。其实Java和JavaScript从字面理解就是Java语言和Java语言的脚本(script脚本)，但是实际上它们并没有什么关系， 如果硬要说关系那就是北大和北大青鸟，雷锋和雷峰塔的关系。

JavaScript的发展

JavaScript一般都被默认为浏览器开发语言，但是事实上上一开始JavaScript也是可以运行到后台的。1994年网景公司就推出Netscape Enterprise Server的服务器端脚本实现，叫LiveWire(服务端的JavaScript脚本语言)。 1997年微软在其服务器IIS3.0支持JSscript(和JavaScript兼容)。但是与当时风靡PHP，Java，.Net相比，并不占有任何优势而日渐式微。

第一次转折期便是Google推出ajax技术，实现无刷新页面更新，极大推动当时web发展同时也使得web端JavaScript发展。随后各个浏览器厂家FireFox, IE, opera，safari用各自技术实现ajax，也掀开浏览器端的军备竞赛，令javaScript性能不断提升。但是因而不同厂家不同技术造成早期军阀割据界面，当时web端开发最头疼就是各种浏览器的各种兼容问题。

而后jQuery出现统一了这个问题，从此也就开始jQuery王朝。下图这个梗说的就是jQuery的在当时的地位。记得2015-2016年前端面试基本都是要求，熟悉jQuery用法，会根据需求开发相应的jQuery插件。而当时基本无论是前端还是后台都是界面开发都是找各种jQuery插件。在这段时间JavaScript发展十分缓慢，1999年12月，ECMAScript 3.0版发布(可以理解为规范)，成为JavaScript的通行标准，得到了广泛支持。但是后面因为利益问题一直得不到相关推进，2008年本应该规范成ECMAScript4.0方案因为分歧太大，只是对原来3.0方案小修小补，后面更为ECMAScript 3.1， 再后来更名为ECMAScript 5。而这个方案在各浏览器厂家成为规范是2011年,所谓的ECMAScript 5.1。所以这段时间内容大部分前端开发不受重视，大多数前端开发是做静态界面和交互，然后给后台更改模板引擎，此时大多数民间人士称为页面仔，切图仔。

第二次转折期，2009 年 Node 的诞生，node官方文档介绍，Node.js是一个基于 Chrome V8 引擎的 JavaScript 运行时。 Node.js 使用高效、轻量级的事件驱动、非阻塞 I/O 模型高性能服务器。而现在看来无论再怎么夸奖它的功劳和贡献不为过，它真正把前端开发从过去手工作业带到现代工业化时代，JavaScript不在锁在浏览器这个沙箱里面玩泥巴，它竟然能赋予前端开发操纵系统魔力。在经过好几年发展nodejs带来稳定版本，甚至被当时前端人员狂吹彩虹屁，号称统一web开发前后台，而且作为后台性能出色轻易解决java这些语言的高并发性能问题。当然这些吹捧过于夸张， 但也可以侧面证明nodejs已经能作为开发主流服务器(取代java有点夸张了，取代php的话还是可能的)

与此同时， angularjs， react js 乃至后面vuejs等mvvn框架出现， es6标准推出，gulp，webpack技术使得前端开发开始往后台开发看齐，愈发规范和标准。HTML5技术推出，使得在前端在移动端将有着极大的想象空间，最典型的应用前景是谷歌的浏览器OS概念——既浏览器取代操作系统，所有软件、APP都运行在云端，用户通过浏览器获取软件服务(移动端刚开始是原生开发，后面后H5出现开始趋向混合开发)。趁着手机发展的红利期，公众号，小程序，webAPP，react-native技术出现，使得市场愈发青睐于前端构建的生态。前端从前台渗透后台同时持续开始往移动端开发渗透。不久web-native技术electron出现，前端也能进行桌面级软件的开发。浏览器支持WEBGL,Canvas使得js也能进行游戏开发,比如Three.js是一个广泛应用并且功能强大的JavaScript 3D库，从创建简单的3D动画到创建交互的3D游戏， 它都能实现。又比如白鹭引擎，现在很多手机上小程序小游戏基于此引擎开发。前端不再有技术瓶颈，而是突然你发现你根本学不动了。

随着市场需求发展，前端一直被人诟病语法结构过于灵活松散，无法构建大型项目也被慢慢改变，Typescript微软出品的誉为JavaScript的超集，在js基础上增加静态语言特性，js的静态语言版本。其他所有静态语言的语法和格式继承同时还保留js特性，使得JavaScript同样具备开发大型项目能力，比如我的编辑器vscode就是用Typescript开发。

总结

前端发展日新月异，处处生机勃勃，每天都涌现出不同新的技术。前端请重新定义不再是单纯页面开发，而是以此技术栈发展起来的前后端统一解决方案体系。这个体系更加贴近用户端，同时也能力根据用户体验延展下去解决方案，也使得中小企业以更低成本开发市场。5G技术已经到来，会有更多更炫酷的动画被加入进来，用户追求更加酷炫3D视觉，比如今天的二手车软件的3D看车，租房软件的3D看房，AR增强现实，VR虚拟现实 技术都将会被加入。到时候会有更多的AR游戏领域的崛起，前端发展从未停止!

最后，总结一句话，凡是可以用 JavaScript 来写的应用，最终都会用 JavaScript 来写。
